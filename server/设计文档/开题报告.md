# 个人健康饮食记录系统的设计与实现 - 开题报告

## 1. 本课题所涉及的问题在国内（外）的研究现状综述

随着社会经济的快速发展和生活水平的不断提高，人们的饮食结构发生了显著变化。高热量、高脂肪、高糖分的食物摄入增加，加之久坐少动的生活方式日益普遍，导致超重、肥胖、糖尿病、高血压等慢性疾病的发病率逐年攀升。根据《中国居民营养与慢性病状况报告（2020年）》数据显示，我国成年居民超重率为34.3%，肥胖率为16.4%，慢性病已成为影响国民健康的重要公共卫生问题。在此背景下，个人健康饮食管理受到越来越多的关注，饮食记录与营养分析系统的研究具有重要的现实意义。

从国外研究现状来看，健康饮食管理应用起步较早，发展相对成熟。美国的MyFitnessPal、Lose It!等应用拥有庞大的食物数据库和活跃的用户社区，支持条形码扫描、餐厅菜单搜索等便捷录入方式。欧洲的Yazio、Lifesum等应用则更注重个性化推荐，结合用户的健康目标提供定制化的饮食方案。近年来，随着人工智能技术的发展，国外研究开始将机器学习、深度学习等技术应用于饮食图像识别、营养素预测等领域。例如，Google的Im2Calories项目尝试通过图像识别自动估算食物热量；麻省理工学院的研究团队开发了基于深度学习的食物识别系统，能够识别超过2000种食物。此外，大语言模型（LLM）的兴起为智能饮食建议提供了新的技术路径，ChatGPT、Claude等模型展现出在营养咨询领域的应用潜力。

在国内，健康饮食管理应用的发展也在加速推进。薄荷健康、Keep、华为健康等应用已积累了大量用户，提供基于《中国食物成分表》的本土化营养数据。学术研究方面，国内学者在饮食记录系统、营养计算算法、健康推荐系统等领域开展了广泛研究。清华大学、浙江大学等高校的研究团队在食物图像识别、个性化营养推荐等方向取得了重要进展。然而，现有系统仍存在一些不足：一是食物数据库不够完善，对中式菜肴的覆盖不足；二是营养分析功能较为单一，缺乏对用户长期饮食趋势的深入分析；三是智能建议功能有限，难以提供真正个性化的饮食指导。

综上所述，个人健康饮食记录系统在国内外均有广泛的研究基础和应用实践，但在本土化数据支持、智能化分析建议等方面仍有较大的改进空间。本课题旨在设计并实现一套功能完善、操作便捷的个人健康饮食记录系统，充分利用《中国食物成分表》数据，并集成大语言模型提供智能化饮食建议，为用户的健康饮食管理提供有效支持。

## 3. 本课题需要重点研究的、关键的问题及解决的思路

### 3.1 营养数据的准确性与完整性

**问题描述：** 食物营养计算的准确性直接影响系统的实用价值，需要建立完善的食物营养数据库。

**解决思路：** 以《中国食物成分表》为基础数据源，该表收录了1700余种常见食物的详细营养数据（热量、蛋白质、脂肪、碳水化合物、膳食纤维、维生素、矿物质等）。通过CSV数据导入脚本，将数据标准化后存入数据库，并建立完善的分类体系，支持后续扩展。

### 3.2 个性化热量目标的计算

**问题描述：** 不同用户的身体状况、活动水平、健康目标各异，如何为每位用户计算合理的每日热量摄入目标。

**解决思路：** 采用国际通用的Mifflin-St Jeor公式计算基础代谢率（BMR），结合用户的活动水平系数和健康目标（减脂/维持/增肌）系数，得出个性化的每日推荐热量。同时，系统保存用户身体状态快照，支持追踪历史变化。

### 3.3 AI智能建议的可控性与安全性

**问题描述：** 大语言模型生成的内容存在不确定性，如何确保AI建议的合理性，避免误导用户。

**解决思路：** 
- 构建结构化Prompt模板，将用户的BMI、今日摄入、健康目标等数据注入上下文，引导模型生成针对性建议
- 设置每日调用次数限制，防止滥用
- 所有AI输出均附带免责声明
- 管理端可配置AI开关和Prompt模板，便于调控

### 3.4 数据一致性与历史追溯

**问题描述：** 用户的身体数据可能发生变化，如何确保历史饮食记录与当时的身体状态保持一致。

**解决思路：** 采用"身体状态快照"设计，每当用户更新身体档案时，系统自动创建一条快照记录。饮食记录通过body_record_id字段关联对应的身体快照，实现历史数据的精准追溯。
