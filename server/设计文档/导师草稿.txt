一、核心功能模块（精准匹配你的需求）
1. 用户模块（基础支撑）
这是系统的入口，核心是获取用户的基础身体数据，为后续计算提供依据：
•	用户注册 / 登录：手机号 / 邮箱 + 密码登录（密码加密存储），支持记住密码、找回密码；
•	个人身体数据管理：
o	录入 / 修改身高、体重、年龄、性别、日常活动量（久坐 / 轻度运动 / 中度运动 / 高强度运动）；
o	自动计算 BMI 值，并根据中国 BMI 标准（<18.5 偏瘦、18.5-23.9 正常、24.0-27.9 超重、≥28 肥胖）判定体型；
o	根据身高、体重、活动量，自动计算每日推荐热量摄入量（基础代谢率 BMR + 活动消耗，公式：
	男性 BMR = 10× 体重 (kg) + 6.25× 身高 (cm) - 5× 年龄 + 5；
	女性 BMR = 10× 体重 (kg) + 6.25× 身高 (cm) - 5× 年龄 - 161；
	每日推荐热量 = BMR × 活动系数（久坐 1.2、轻度 1.375、中度 1.55、高强度 1.725））。
2. 食物库模块（核心数据支撑）
系统能计算热量的前提，需要内置标准化的食物热量数据：
•	基础食物库：预设常见食物（米饭、馒头、鸡蛋、鸡胸肉、苹果、牛奶等），包含每 100 克的热量、蛋白质、碳水、脂肪；
•	食物库管理（管理员端）：支持新增 / 修改 / 删除食物信息，保证食物数据的准确性和丰富性。
3. 饮食记录模块（核心操作模块）
用户记录每日饮食，是系统计算热量的来源：
•	饮食录入：
o	选择餐次（早餐 / 午餐 / 晚餐 / 加餐）；
o	搜索并选择食物库中的食物，输入食用分量（克）；
o	支持单次录入多种食物，自动累加分量和热量；
•	饮食记录查询：按日期查看当日 / 历史饮食记录，显示每种食物的分量、热量占比；
•	记录修改 / 删除：修正错误录入的食物或分量。
4. 热量分析与反馈模块（核心价值模块）
这是系统的核心输出，直接匹配你 “热量超标提醒 + 饮食调整建议” 的需求：
•	实时热量计算：用户录入饮食后，自动累加当日总热量摄入，对比 “每日推荐热量摄入量”，显示 “已摄入占比”“剩余可摄入热量”；
•	超标预警：若当日总热量超过推荐值，系统高亮提示 “热量超标”，并拆解：
o	具体超标营养素（如 “脂肪摄入超标 30%”“碳水摄入超标 20%”）；
o	超标食物来源（如 “今日午餐的炸鸡贡献了 60% 的超标脂肪”）；
•	个性化饮食调整建议：
o	针对 BMI 状况 + 热量超标类型给出建议（如 “你 BMI 为 25（超重），今日脂肪摄入超标，建议晚餐替换为清蒸鱼 + 西兰花，减少油炸食品摄入”；“你 BMI 为 18（偏瘦），今日热量摄入不足，建议加餐 1 个鸡蛋 + 200ml 牛奶”）；
o	推荐适配的低热量 / 高营养食物（如超标时推荐芹菜、鸡胸肉、燕麦；不足时推荐坚果、全脂牛奶、红薯）。
5. 数据可视化模块（辅助感知）
让用户直观看到自身热量摄入和 BMI 状况，提升使用体验：
•	当日热量分布：用饼图展示早 / 中 / 晚 / 加餐的热量占比；
•	BMI 与热量趋势：用折线图展示近 7 天的 BMI（若体重更新）、每日热量摄入变化；
•	超标记录汇总：按周统计热量超标的天数、主要超标营养素，辅助用户了解自身饮食问题。
