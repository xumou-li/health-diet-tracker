## 二、功能模块设计

### 2.1 用户端功能模块

#### 模块1: 用户账户与身体档案

**保留功能**：
- 手机号/邮箱注册登录
- 密码找回（验证码重置）
- 身体档案维护（性别、出生日期、身高、体重、活动水平）
- 自动计算 BMI、BMR、每日推荐热量

**优化点**：
- 出生日期替代年龄（支持精确计算，原设计已包含）
- 增加"健康目标"设置：减脂/增肌/维持体重，影响热量目标系数
- 增加"饮食偏好"标签：素食/清淡/无麻辣等（用于AI推荐）

**新增功能**：
- 微信一键登录（UniApp原生支持，简化登录流程）
- 体重变化提醒（连续7天未更新体重时推送）

---

#### 模块2: 饮食记录

**保留功能**：
- 选择日期、餐次（早/中/晚/加餐）
- 从食物库搜索食物
- 输入食用重量（克）
- 支持一次添加多个食物
- 编辑/删除记录
- 日历视图查看历史

**优化点**：
- 增加"快捷记录"：复制昨天某餐、常用组合（如"早餐标配"）
- 搜索优化：支持拼音首字母、模糊匹配、分类筛选
- 重量输入优化：提供常用单位快捷选择（1碗≈200g、1个≈50g等）

**新增功能**：
- "我的食谱"：用户可保存常吃的食物组合，一键添加
- 语音输入（调用UniApp原生能力，识别"一碗米饭"自动匹配）

---

#### 模块3: 食物库交互

**保留功能**：
- 分类浏览（基于中国食物分类表的二级分类）
- 搜索框（模糊匹配）
- 食物详情页（每100g营养数据）
- 常用食物收藏夹

**优化点**：
- 利用CSV数据的完整营养素信息（不仅是四大营养素，还有维生素、矿物质等）
- 食物详情页展示更丰富信息：膳食纤维、胆固醇、钙、铁等

**新增功能**：
- "相似食物推荐"：查看某食物时，显示同类低热量替代品
- 热门食物排行（基于全站用户使用频次）

---

#### 模块4: 营养摄入统计与反馈

**保留功能**：
- 当日汇总面板（总热量vs推荐值）
- 三大营养素实际摄入量与占比
- 超标提示

**优化点**：
- 超标提示细化：区分"轻度超标(100%-120%)"、"严重超标(>120%)"
- 增加不足提示：热量<80%、蛋白质不足等

**新增功能**：
- 营养素雷达图：直观展示各营养素摄入与推荐值的对比
- 膳食平衡评分：综合打分（0-100分），让用户一眼看出今天吃得好不好
- 周/月报告：汇总一段时间的饮食数据，生成简报

---

#### 模块5: 数据可视化

**保留功能**：
- 日视图：热量进度条、营养素环形图、餐次热量分布
- 周趋势：7天热量变化折线图、达标天数统计

**优化点**：
- 移动端适配：使用uCharts替代ECharts，更适合小程序/App

**新增功能**：
- 体重趋势图：展示体重变化曲线
- 热量收支图：每日热量摄入 vs BMR消耗
- 营养素趋势：某段时间内蛋白质/脂肪/碳水的摄入变化

---

#### 模块6: AI智能建议

**原设计**：规则引擎 + 可选LLM  
**调整为**：强依赖LLM（根据用户选择）

**功能设计**：
- **每日饮食评价**：根据当日饮食数据，LLM生成个性化评价
- **饮食建议**：基于BMI、健康目标、近期摄入趋势，推荐下一餐吃什么
- **食物替换建议**：某食物热量过高时，推荐低热量替代品
- **问答功能**：用户可以问"我今天还能吃什么？""这周蛋白质够吗？"

**技术实现**：
- 接入 DeepSeek / OpenAI / 通义千问 等LLM API
- 构建结构化Prompt模板，注入用户数据（BMI、今日摄入、健康目标等）
- 限流控制：每用户每天10次（可配置）
- 免责声明："仅供参考，不替代专业医疗建议"

**Prompt示例**：
```
你是一个专业的营养师助手。
用户信息：性别{gender}，BMI={bmi}({bmi_status})，健康目标={goal}
今日已摄入：热量{calorie}kcal（目标{target}kcal），蛋白质{protein}g，脂肪{fat}g，碳水{carb}g
请给出简短的饮食建议（50字以内）。
```

### 2.2 管理端功能模块

#### 模块1: 管理员账户管理
- 管理员登录（独立账户体系，非用户表）
- 角色权限：超级管理员 / 普通管理员
- 操作日志查看

#### 模块2: 用户管理
- 用户列表（ID、注册时间、最近活跃、BMI分布）
- 用户详情查看（脱敏展示）
- 冻结/解冻账号
- 用户统计看板：注册趋势、活跃度分布

#### 模块3: 食物数据库管理
- **CRUD操作**：增删改食物信息
- **批量导入**：CSV文件上传，自动解析入库
- **分类管理**：维护一级/二级分类
- **审核机制**：新增食物需审核后生效
- **数据校验**：检测异常数据（如热量为负、数值过大）

#### 模块4: 系统配置
- AI功能开关（全局）
- AI调用限额配置
- LLM API配置（API Key、模型选择）
- Prompt模板管理
- 系统公告管理

#### 模块5: 数据统计与分析
- 全站数据概览：用户总数、今日活跃、饮食记录数
- 食物使用排行：哪些食物被记录最多
- 用户BMI分布：偏瘦/正常/超重/肥胖人数统计
- AI调用统计：每日调用次数、成功率

## 三、数据库设计

### 3.1 完整表结构

#### 1. users（用户表）
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    phone VARCHAR(20) UNIQUE COMMENT '手机号',
    email VARCHAR(100) UNIQUE COMMENT '邮箱',
    password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希',
    gender TINYINT NOT NULL COMMENT '0=男, 1=女, 2=其他',
    birthday DATE NOT NULL COMMENT '出生日期',
    height_cm INTEGER NOT NULL COMMENT '身高(cm)',
    weight_kg DECIMAL(5,2) NOT NULL COMMENT '体重(kg)',
    activity_level TINYINT NOT NULL COMMENT '1=久坐, 2=轻度, 3=中度, 4=高强度',
    health_goal TINYINT DEFAULT 1 COMMENT '1=维持, 2=减脂, 3=增肌',
    diet_preference VARCHAR(100) COMMENT '饮食偏好标签,逗号分隔',
    bmr INTEGER NOT NULL COMMENT '基础代谢率(kcal/天)',
    daily_calorie_goal INTEGER NOT NULL COMMENT '每日推荐热量(kcal)',
    bmi DECIMAL(4,2) NOT NULL COMMENT 'BMI值',
    wechat_openid VARCHAR(100) COMMENT '微信OpenID',
    last_weight_update DATE COMMENT '最近体重更新日期',
    is_frozen TINYINT(1) DEFAULT 0 COMMENT '是否冻结',
    is_deleted TINYINT(1) DEFAULT 0 COMMENT '软删除标志',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. body_records（身体状态快照表）
```sql
CREATE TABLE body_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL COMMENT '用户ID',
    height_cm INTEGER NOT NULL COMMENT '身高(cm)',
    weight_kg DECIMAL(5,2) NOT NULL COMMENT '体重(kg)',
    age INTEGER NOT NULL COMMENT '记录时的年龄',
    gender TINYINT NOT NULL COMMENT '0=男,1=女,2=其他',
    activity_level TINYINT NOT NULL COMMENT '1=久坐,2=轻度,3=中度,4=高强度',
    bmi DECIMAL(4,2) NOT NULL COMMENT 'BMI值',
    bmr INTEGER NOT NULL COMMENT '基础代谢率(kcal/天)',
    daily_calorie_goal INTEGER NOT NULL COMMENT '当日推荐总热量(kcal)',
    recorded_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '快照时间'
);
CREATE INDEX idx_body_user_date ON body_records(user_id, DATE(recorded_at));
```

#### 3. food_categories（食物分类表）
```sql
CREATE TABLE food_categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code VARCHAR(10) NOT NULL UNIQUE COMMENT '分类编码',
    parent_code VARCHAR(10) COMMENT '父级编码，一级分类为空',
    name VARCHAR(50) NOT NULL COMMENT '分类名称',
    sort_order INTEGER DEFAULT 0 COMMENT '排序'
);
```

#### 4. foods（食物表）
```sql
CREATE TABLE foods (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '食物名称',
    category_code VARCHAR(10) COMMENT '一级分类编码',
    sub_category_code VARCHAR(10) COMMENT '二级分类编码',
    edible_portion INTEGER DEFAULT 100 COMMENT '可食部分(%)',
    calorie_per_100g INTEGER NOT NULL COMMENT '每100g热量(kcal)',
    protein_per_100g DECIMAL(5,2) NOT NULL COMMENT '蛋白质(g/100g)',
    carb_per_100g DECIMAL(5,2) NOT NULL COMMENT '碳水(g/100g)',
    fat_per_100g DECIMAL(5,2) NOT NULL COMMENT '脂肪(g/100g)',
    fiber_per_100g DECIMAL(5,2) COMMENT '膳食纤维(g/100g)',
    cholesterol_per_100g DECIMAL(5,2) COMMENT '胆固醇(mg/100g)',
    sodium_per_100g DECIMAL(5,2) COMMENT '钠(mg/100g)',
    calcium_per_100g DECIMAL(5,2) COMMENT '钙(mg/100g)',
    iron_per_100g DECIMAL(5,2) COMMENT '铁(mg/100g)',
    vitamin_c_per_100g DECIMAL(5,2) COMMENT '维生素C(mg/100g)',
    is_approved TINYINT(1) DEFAULT 1 COMMENT '是否审核通过',
    created_by INTEGER COMMENT '创建者管理员ID',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_foods_category ON foods(category_code);
CREATE INDEX idx_foods_name ON foods(name);
```

#### 5. favorite_foods（用户收藏食物表）
```sql
CREATE TABLE favorite_foods (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL COMMENT '用户ID',
    food_id INTEGER NOT NULL COMMENT '食物ID',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE UNIQUE INDEX idx_favorite_user_food ON favorite_foods(user_id, food_id);
```

#### 6. user_recipes（用户食谱表）
```sql
CREATE TABLE user_recipes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL COMMENT '用户ID',
    name VARCHAR(50) NOT NULL COMMENT '食谱名称',
    items TEXT NOT NULL COMMENT 'JSON数组:[{food_id, weight_g}, ...]',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_recipes_user ON user_recipes(user_id);
```

#### 7. meal_records（饮食记录表）
```sql
CREATE TABLE meal_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL COMMENT '用户ID',
    food_id INTEGER NOT NULL COMMENT '食物ID',
    body_record_id INTEGER NOT NULL COMMENT '对应的身体快照ID',
    date DATE NOT NULL COMMENT '用餐日期',
    meal_type TINYINT NOT NULL COMMENT '1=早餐, 2=午餐, 3=晚餐, 4=加餐',
    weight_g DECIMAL(7,2) NOT NULL COMMENT '实际摄入重量(克)',
    calorie INTEGER NOT NULL COMMENT '实际热量(kcal)',
    protein DECIMAL(6,2) NOT NULL COMMENT '实际蛋白(g)',
    carb DECIMAL(6,2) NOT NULL COMMENT '实际碳水(g)',
    fat DECIMAL(6,2) NOT NULL COMMENT '实际脂肪(g)',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_meals_user_date ON meal_records(user_id, date);
CREATE INDEX idx_meals_body_record ON meal_records(body_record_id);
```

#### 8. ai_suggestions（AI建议记录表）
```sql
CREATE TABLE ai_suggestions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL COMMENT '用户ID',
    suggestion_type VARCHAR(20) NOT NULL COMMENT '建议类型: suggestion/evaluate/recommend/chat',
    prompt TEXT COMMENT '发送给LLM的Prompt',
    content TEXT NOT NULL COMMENT 'AI生成的建议文本',
    model VARCHAR(50) COMMENT '使用的模型名称',
    tokens_used INTEGER COMMENT 'Token消耗量',
    response_time_ms INTEGER COMMENT '响应时间(毫秒)',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_ai_user_date ON ai_suggestions(user_id, DATE(created_at));
```

#### 9. admins（管理员表）
```sql
CREATE TABLE admins (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希',
    role TINYINT DEFAULT 1 COMMENT '1=普通管理员, 2=超级管理员',
    is_active TINYINT(1) DEFAULT 1 COMMENT '是否启用',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_login_at DATETIME COMMENT '最后登录时间'
);
```

#### 10. admin_logs（管理员操作日志表）
```sql
CREATE TABLE admin_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    admin_id INTEGER NOT NULL COMMENT '管理员ID',
    action VARCHAR(50) NOT NULL COMMENT '操作类型',
    target_type VARCHAR(30) COMMENT '目标类型: user/food/config等',
    target_id INTEGER COMMENT '目标ID',
    details TEXT COMMENT '操作详情JSON',
    ip_address VARCHAR(50) COMMENT 'IP地址',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_logs_admin ON admin_logs(admin_id);
CREATE INDEX idx_logs_date ON admin_logs(DATE(created_at));
```

#### 11. system_configs（系统配置表）
```sql
CREATE TABLE system_configs (
    id INTEGER PRIMARY KEY DEFAULT 1,
    ai_enabled TINYINT(1) DEFAULT 1 COMMENT 'AI功能开关',
    ai_daily_limit INTEGER DEFAULT 10 COMMENT '每用户每日AI调用限额',
    ai_model VARCHAR(50) DEFAULT 'deepseek-chat' COMMENT '默认AI模型',
    ai_api_key VARCHAR(255) COMMENT 'AI API密钥(加密存储)',
    ai_prompt_template TEXT COMMENT 'AI提示词模板',
    announcement TEXT COMMENT '系统公告',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 六、核心业务逻辑

### 6.1 BMI/BMR计算公式

```python
# BMI计算
def calculate_bmi(weight_kg, height_cm):
    height_m = height_cm / 100
    return round(weight_kg / (height_m ** 2), 2)

# BMI状态判定（中国标准）
def get_bmi_status(bmi):
    if bmi < 18.5:
        return "偏瘦"
    elif bmi < 24:
        return "正常"
    elif bmi < 28:
        return "超重"
    else:
        return "肥胖"

# BMR计算（Mifflin-St Jeor公式）
def calculate_bmr(weight_kg, height_cm, age, gender):
    if gender == 0:  # 男性
        return int(10 * weight_kg + 6.25 * height_cm - 5 * age + 5)
    else:  # 女性
        return int(10 * weight_kg + 6.25 * height_cm - 5 * age - 161)

# 每日推荐热量
def calculate_daily_calorie(bmr, activity_level, health_goal):
    # 活动系数
    activity_factors = {1: 1.2, 2: 1.375, 3: 1.55, 4: 1.725}
    # 健康目标系数
    goal_factors = {1: 1.0, 2: 0.85, 3: 1.15}  # 维持/减脂/增肌
    
    base = bmr * activity_factors.get(activity_level, 1.2)
    return int(base * goal_factors.get(health_goal, 1.0))
```

### 6.2 营养素推荐比例

```
碳水化合物: 50-65% 的总热量
蛋白质: 10-20% 的总热量  
脂肪: 20-30% 的总热量

换算公式（基于每日推荐热量 X kcal）:
- 碳水推荐量(g) = X * 0.55 / 4  (1g碳水=4kcal)
- 蛋白推荐量(g) = X * 0.15 / 4  (1g蛋白=4kcal)
- 脂肪推荐量(g) = X * 0.25 / 9  (1g脂肪=9kcal)
```

### 6.3 膳食平衡评分算法

```python
def calculate_diet_score(actual, target):
    """
    计算膳食平衡评分 (0-100分)
    actual: {calorie, protein, carb, fat}
    target: {calorie, protein, carb, fat}
    """
    scores = []
    
    # 热量评分 (权重40%)
    calorie_ratio = actual['calorie'] / target['calorie']
    if 0.9 <= calorie_ratio <= 1.1:
        calorie_score = 100
    elif 0.8 <= calorie_ratio <= 1.2:
        calorie_score = 80
    else:
        calorie_score = max(0, 100 - abs(calorie_ratio - 1) * 100)
    scores.append(('calorie', calorie_score, 0.4))
    
    # 三大营养素评分 (各权重20%)
    for nutrient in ['protein', 'carb', 'fat']:
        ratio = actual[nutrient] / target[nutrient] if target[nutrient] > 0 else 0
        if 0.8 <= ratio <= 1.2:
            score = 100
        else:
            score = max(0, 100 - abs(ratio - 1) * 80)
        scores.append((nutrient, score, 0.2))
    
    # 加权总分
    total = sum(score * weight for _, score, weight in scores)
    return round(total)
```
